# Coding Test Submission

## Project Structure

- `A/` - Solution for Question A
- `B/` - Solution for Question B
- `C/` - Written answers for the Xero API questions
- `frontend/` - E-commerce Product Detail Page implementation

---

## How to Run

### A and B

Run the Python files:

```bash
python A/solution.py
python B/solution.py
```

### C

Open the Markdown file:

```text
C/xero_api_answers.md
```

### Frontend

Open the following file directly in a browser:

```text
frontend/index.html
```

Alternatively, you may use VS Code with the Live Server extension to run the frontend locally.

---

## Assumptions

- Questions A and B follow standard input/output requirements.
- Xero API answers are based on Xero Accounting API documentation and standard REST API integration practices.
- The frontend implementation uses mock APIs instead of a real backend service.
- Product data is hardcoded for demonstration purposes.
- Cart data is stored in frontend memory only.
- No external database or backend server is required.
